## What This Repo Is

A **Docusaurus 3** static documentation portal for Peepul Farm's IoT platform, deployed on **Cloudflare Pages** at `techinfo.peepul.farm`. It serves four products: **iDhara**, **APFC**, **ORC**, and **DemeterCloud**.

## Commands

```bash
npm start          # local dev server (hot reload)
npm run build      # production build → ./build/
npm run serve      # serve the built output locally
npm run clear      # clear Docusaurus cache if builds behave oddly
```

## Cloudflare Pages Deployment

- **Build command:** `npm run build`
- **Output directory:** `build`
- **Node version:** 20

## Architecture

```
docusaurus.config.js   — site config (URL, navbar, footer, theme)
sidebars.js            — four named sidebars (one per product)
src/pages/index.js     — custom home page with product cards
src/css/custom.css     — Peepul Farm green theme overrides
docs/
  idhara/             — iDhara product docs
  apfc/               — APFC product docs
  orc/                — ORC product docs
  demetercloud/       — DemeterCloud platform docs
static/img/           — logo and favicon
```

Each product folder contains these sub-sections (as applicable):
`release-notes/`, `user-manual/`, `api/`, `firmware/`, `mobile/`, `admin/`

Sidebar entries in `sidebars.js` use `autogenerated` from those sub-directories — new `.md` files appear automatically in the correct sidebar.

## Webpack Pinning

`package.json` overrides webpack to `5.95.0` because webpack 5.106+ breaks `webpackbar` (used internally by Docusaurus). Do not remove this override.

## Adding Documentation

1. Create a `.md` file in the appropriate `docs/<product>/<section>/` directory.
2. Add frontmatter:
   ```md
   ---
   title: My Page Title
   sidebar_position: 2
   ---
   ```
3. The file appears automatically in the sidebar — no changes to `sidebars.js` needed for existing sections.
4. To add a **new section** to a product, create the directory, add docs inside it, and add a `category` entry in `sidebars.js`.
