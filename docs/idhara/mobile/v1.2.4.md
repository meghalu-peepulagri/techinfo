## What's New

### Motor & Fault Management

- **Motor Settings** — Updated the Motor Settings screen title for clarity.
- **Clear Fault** — Added a "Clear Fault" option that appears when the motor is turned on.
- **Fault Enable/Disable** — Users can now enable or disable individual faults.
- **Fault Settings Screen** — Redesigned to show on/off status and related information for each fault.

### Dashboard

- **Device Runtime** — Each device card footer now displays the device's runtime.
- **Fault Status** — Fault status is shown on the device card in the dashboard.
- **Clear Faults from Dashboard** — Faults can now be cleared directly from the dashboard.
- **Last Motor Activity** — The last activity of the motor state is displayed on the dashboard.

### Informational

- **Mode Information** — Added information about Auto and Manual modes.
- **Settings Information** — Added details about FLC, overload, and dry run settings, as well as fault settings (on/off).
- **Temporary Off Warning** — A warning message is shown when a motor is temporarily turned off:
  > "This is temporarily turned off. The motor will turn on after a power recycle."

  :::note Pending Clarification
  Exact trigger condition and behavior for this warning needs confirmation.
  :::

### UX & Navigation

- **Settings as Separate Menu Item** — Settings has been moved to its own menu item in the navigation.
- **Device Image Capture** — Users can now capture a device image while adding a new device.

### Environments

- **Dev / Prod App Separation** — Separate app builds are now maintained for Development and Production environments.
