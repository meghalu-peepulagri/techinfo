# IoT Internal Documentation Portal Setup

## Objective

Create an internal documentation portal for an IoT platform used by multiple engineering teams.  
The portal should be easy to maintain, structured clearly, and hosted as a static documentation website.

The documentation will be used internally by:

- Cloud / Backend Team
- Firmware / Embedded Team
- Mobile Development Team
- Admin Dashboard Team
- QA Team
- Product / Field Engineers

The documentation system should support:

- Release Notes
- API Documentation
- Firmware Documentation
- Mobile App Documentation
- Admin Dashboard Documentation
- User Manuals

The final output should be a documentation website built using **Docusaurus**, stored in a **GitHub repository**, and deployed on **Cloudflare Pages**.
